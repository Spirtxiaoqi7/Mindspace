# 安全说明

## 报告安全问题

请优先通过 GitHub Security Advisory 私下报告漏洞，不要在公开 Issue 中提交可利用细节、真实 API Key、用户档案、对话记录或声音素材。

报告中建议包含：

- 受影响版本；
- 可复现步骤；
- 预期与实际行为；
- 是否涉及远程代码执行、路径穿越、密钥暴露或用户数据；
- 已完成的最小验证。

## 密钥和用户数据

Mindspace 的 API Key、更新签名私钥、会话、档案、数据库、日志、模型和音频均位于被 `.gitignore` 排除的目录。提交前仍应执行敏感信息扫描，因为忽略规则不能替代审计。

如果密钥曾进入 Git 历史，单纯删除文件并不安全；应立即撤销密钥，并重写受影响的 Git 历史。
