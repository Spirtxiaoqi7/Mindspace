# Mindspace 产品介绍与用户指南

## 认识 Mindspace

Mindspace 是一款面向长期陪伴、角色对话和个人知识管理的 Windows AI 应用。它不只生成一条回复，还会在用户允许的范围内维持人物设定、关系状态、偏好和连续对话，让不同时间发生的交流保持前后一致。

产品采用桌面启动器和本地数据工作区。文字聊天、知识库、记忆管理、人物头像、实时语音和运行诊断集中在同一个界面中；普通用户不需要安装 Python、Git、PowerShell 7、Node.js 或开发工具。

## 你可以用它做什么

### 连续的角色对话

- 为 AI 设置名称、性格、说话方式、关系定位和行为边界。
- 使用自定义用户头像与 AI 头像，聊天、人物卡和语音页面共享同一配置。
- 让 AI 根据当前关系和上下文主动说点什么，无需先输入一段完整问题。
- 在文字模式和实时语音模式之间切换，已有会话和消息不会丢失。

Mindspace 的基础规则会明确告诉 AI：情感与互动通过文字及括号中的语气表达，不把虚构的实体接触当作现实发生的事情。

### 可检查、可修改的长期记忆

Mindspace 将记忆分为两部分：原始对话用于理解语境，结构化记忆用于保存经过确认的人物事实和状态。

- 用户当前明确表达，才是常规修改人物档案的主要依据。
- 当前人物档案中的 JSON 是其字段范围内的最高持久事实。
- 历史、召回内容和知识库可以辅助理解，但不能自行覆盖人物档案。
- “喜欢”和“不喜欢”等对立信息会按同一实体自动消除冲突。
- 可在“记忆中心”查看、修改、删除和恢复结构化记忆。

删除某条 AI 回复后，这条回复会立即退出页面和召回，但人物档案不会在删除瞬间被强行改写。下一次正常对话会携带删除事件，让系统结合剩余历史重新校正。

### 个人知识库与可见引用

可以粘贴或上传 TXT、Markdown、JSON 资料。系统会把长文本拆成适合检索的子块，同时保留较完整的父级语境。

回答时，词法检索和语义向量检索会共同寻找相关内容。聊天页的“本轮引用”会显示真正进入本轮上下文的知识或记忆，而不是只显示一个无法解释的相关度数字。

如果本地向量模型不可用，知识检索仍可退回 BM25+ 词法模式；可选精排模型未安装时，会退回 RRF 融合结果，不影响基础聊天。

### 流式文字和实时语音

模型回复会逐字显示，不需要等待整条回复完成。TTS 遇到完整句号、问号、省略号等边界后立即提交一句，同时继续生成下一句。

实时语音页面提供：

- AI 头像主体和头像衍生的虚化背景；
- 当前转写与流式回复字幕；
- 麦克风能量和播放强度反馈；
- 自动断句、自动发送和连续免按住对话；
- 用户再次开口时中断当前生成和朗读；
- 固定退出、失败重试及文字降级能力。

括号中的动作或语气描写会显示在文字中，但不会被 TTS 朗读。

## 第一次使用

### 1. 安装与初始化

在 Windows 10/11 x64 上运行 Mindspace 安装程序。应用采用当前用户安装，不要求管理员权限，也不会修改系统 Python 或系统 PATH。

首次启动时，Launcher 会依次检查并部署私有运行环境、核心依赖和中文向量模型。下载支持进度、取消、失败重试、断点续传和 SHA-256 校验。

本地 ASR 和本地 CosyVoice 属于可选组件。没有兼容 NVIDIA 显卡或驱动时，文字聊天、知识库、LLM API 和云端 TTS 仍可正常使用。

### 2. 配置真实语言模型

初始 `demo` 模式只用于证明界面和 LangGraph 调度可用，会返回固定演示回复。要进行真实对话：

1. 打开“设置 → 模型与角色”。
2. 将运行模式切换为“真实 API（OpenAI 兼容）”。
3. 填写 API 地址、模型名称和 API 密钥。
4. 点击“自检 LLM + TTS API”。
5. 保存设置后开始新一轮对话。

API 密钥只保存在服务端配置文件中；设置查询接口只返回“是否已配置”，不会把密钥明文发回页面。

### 3. 配置角色和头像

在“模型与角色”中填写用户称呼、角色名称、用户设定和角色系统提示。在“头像”页分别上传用户与 AI 图片，可调整比例、缩放和位置。

人物设定前三轮会进入快速补齐窗口：当持久人物字段仍有较多空缺时，系统只能从用户设定、角色设定或当前明确输入中补齐可验证内容；第四轮开始恢复普通的小幅更新规则。

### 4. 选择语音链路

- 云端 TTS：正式联网版默认链路，填写对应 API 配置即可。
- 本地 CosyVoice：需要安装可选模型和运行时，并上传参考音频、识别或校对参考文本。

参考音频和参考文本必须内容一致，否则克隆音色容易出现发音、音色或节奏异常。

## 主界面说明

- 左侧会话栏：新建、选择和管理会话，进入知识库、记忆、人物档案与诊断。
- 中间聊天区：查看消息、流式状态、本轮引用、实时语音和主动回复入口。
- 右侧执行详情：查看实际运行节点、校验状态和本轮召回内容；可以随时关闭。
- 设置页：管理模型、角色、头像、RAG、协议和语音参数。

所有弹窗只通过关闭或取消按钮退出。存在未保存内容时，按 `Escape` 会先请求确认，不会因误点外部空白而丢失编辑。

## 数据、隐私与联网边界

会话、人物档案、结构化记忆、知识库、头像和参考音频保存在当前 Windows 用户的 Mindspace 工作区。卸载时默认保留个人数据，只有明确选择才会同时删除。

需要注意：

- 使用云端 LLM 时，本轮 Prompt 会发送给你配置的模型服务商。
- 使用云端 TTS 时，待朗读文本会发送给 TTS 服务商。
- 本地 ASR/CosyVoice 在本机运行；云端链路遵循对应服务商的隐私政策。
- Mindspace 不会把 API 密钥放进模型 Prompt。
- “本轮引用”和“执行详情”用于帮助用户检查实际发送给模型的上下文来源。

如果处理敏感内容，应选择可信服务商，或只启用具备本地运行条件的链路。

## 更新与恢复

Launcher 会读取官方签名更新目录。日常业务更新通常只下载较小的 Core 包；只有 Launcher 和安装器本身变化时才需要较大的安装器更新。

更新流程会校验 Ed25519 签名、文件大小和 SHA-256，并先部署到 staging 目录。新核心启动失败时会恢复上一版本，不覆盖用户数据、模型或私有运行环境。

下载中断后可以续传；组件异常时可在 Launcher 中执行单组件重试或“修复环境”。

## 常见问题

### 为什么 AI 一直回复“这是由 LangGraph 调度完成的一次确定性演示回复”？

当前仍处于 `demo` 模式。请配置真实 LLM API、切换运行模式并完成自检。

### 为什么本地实时语音不可用？

检查 Launcher 中的 ASR 组件状态、NVIDIA 驱动、麦克风权限以及 `8766` 服务健康状态。缺少本地语音条件不会影响文字聊天。

### 为什么精排显示未就绪？

cross-encoder 精排是可选本地组件。未安装时系统按设计使用 BM25+、向量和 RRF，不代表知识检索完全不可用。

### 删除回复后，记忆为什么没有立即改变？

这是保护机制。删除只表示该回复不应继续作为真实历史，系统会在下一次成功的正常对话中重新评估相关档案，避免仅凭一次删除直接破坏权威信息。

### 如何检查服务是否正常？

打开“系统诊断”，查看主服务、会话、知识、语音、检索和数据根基状态。`foundation.ok=true`、SQLite 为 `ok` 且未完成事务和投影故障均为 0，表示数据层正常。

## 当前能力边界

- LLM、云端 TTS 的质量、限流和价格由所选服务商决定。
- Provider 不返回缓存 usage 时，系统只能标记为 `unreported`，不会伪造缓存命中率。
- 实体别名必须经过人工确认；系统不会仅凭语义相似自动合并两条人物记忆。
- Skill、MCP 和自主工具在 Prompt 尾部保留了扩展位置，但当前不能把预留接口视为已经完成的自主工具系统。
